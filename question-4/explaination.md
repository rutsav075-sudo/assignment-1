# Command Explanations - Question 4

- `lsof -u hritikshukla | head -n 15`: I listed the open files filtered by my username to view files currently being accessed, limiting output to the first 15 lines for readability.
- `ls -la /dev/fd`: I investigated the file descriptors of my current shell environment by listing standard input (0), output (1), and error (2) streams.
- `ls /fake_directory > standard_out.txt 2> error_out.txt`: I intentionally ran a command on a non-existent directory to demonstrate how standard output and standard error are redirected to separate files.
- `cat error_out.txt`: I read the error file to confirm that the error message generated by the failed command was successfully captured and routed via file descriptor 2.
- `ulimit -a`: I displayed all current resource limits for my user shell, including the maximum number of open file descriptors allowed simultaneously.
